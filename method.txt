CLIENT REQUEUST METHOD
<address:int><header/body(int or str)>

address summary
0 - ping
1 - message that client received message when sending file from server hasn't completed
2~ - custom address which developer can specify



SERVER RESPONSE METHOD
<rand> = random number with 6 digits(without encoding)
^ = without encoding
defalut file sending system

If ok&&need to send file:
1st - <rand><status code^><file size:int><file name:str>
2nd ~ <rand>send file data(One cloud variable can store 250 digits of number.)

If don't need to send something
<rand><status code^>

STATUS CODE
0 - rejected
1 - OK
2 - Error
*Status code will be sent WITHOUT encoding.
